# CipherMsg 🦊🔐

End-to-end шифровальщик для любых мессенджеров — сообщений и изображений.  
Всё шифрование локально в браузере. Нет сервера, нет куков, нет логов, нет внешних зависимостей.

## Запуск

```bash
docker compose up -d
```

По умолчанию поднимается на порту `1337`. Открой: `http://localhost:1337`

или попробуй уже готовый запущенный билд

https://foxhardsoftness.ru/criptofox/

## Функциональность

### 🔒 Вкладка «Зашифровать / Расшифровать» — текстовые сообщения

Шифрует произвольный текст в Base64-строку, которую можно отправить в любом мессенджере.

**Бинарный формат пакета (Base64 → байты):**

```
[1 byte]   KDF ID       — 0x01=PBKDF2-SHA-512 / 0x02=PBKDF2-SHA-256
[32 bytes] Salt          — криптослучайный
[4 bytes]  Iterations    — uint32 LE, случайно 1000–10000
[12 bytes] Nonce         — AES-GCM IV, криптослучайный
[N bytes]  Шифротекст    — AES-256-GCM, содержит внутри:
              [6 bytes]  "SUCC0K" — маркер верификации пароля
              [4 bytes]  Длина сообщения uint32 LE
              [M bytes]  UTF-8 текст сообщения
              [16 bytes] GCM auth tag
```

Маркер `SUCC0K` зашифрован вместе с сообщением. Если пароль неверный — AES-GCM выбросит ошибку аутентификации раньше, чем дойдёт до проверки маркера (двойная защита).

### 🖼 Вкладка «Фото» — шифрование изображений

Шифрует пиксели PNG/JPEG и отдаёт валидный PNG, который выглядит как цветной шум.  
Параметры KDF зашиваются в `tEXt` chunk PNG-файла — получателю нужен только пароль.

**Формат метаданных PNG (chunk `tEXt`, key=`CipherMsg`):**

```
chacha20poly1305|<salt_base64>|<nonce_base64>|<iterations>|<orig_fmt>|<poly1305_tag_base64>
```

**Важно:** отправляй зашифрованный файл через «Прикрепить файл», **не как фотографию**.  
Мессенджеры пережимают изображения и стирают метаданные — шифротекст и параметры KDF будут уничтожены.

**Технические детали:**
- Шифруются только **RGB байты** (без alpha-канала) — защита от Canvas premultiplied alpha
- Alpha всегда `255` в зашифрованном PNG — браузер не искажает пиксели при чтении через Canvas
- На выходе всегда PNG, независимо от входного формата (JPEG конвертируется)

## Стек безопасности

| Компонент | Текстовые сообщения | Изображения |
|-----------|--------------------|----|
| Шифр | AES-256-GCM | ChaCha20-Poly1305 |
| KDF | PBKDF2-SHA-512 / SHA-256 | PBKDF2-SHA-512 |
| Соль | 32 байта random | 32 байта random |
| Nonce/IV | 12 байт random | 12 байт random |
| Итерации | 1000–10000 random | 1000–10000 random |
| MAC | GCM auth tag (128 bit) | Poly1305 (128 bit) |
| Библиотеки | WebCrypto API (нативный) | Встроенная реализация RFC 8439 |

Все криптографические операции выполняются в браузере. Никакие данные не отправляются на сервер — nginx отдаёт только статику.
